# 算法平台中的两条 Pipeline

## Pipeline 1：固定来源的新内容采集 Pipeline

(Fixed-source New Content Pipeline)

### 1. Pipeline 定义

**Pipeline 1 的目标是：**

> 从一组**事先确定的高质量、固定来源**中，持续采集“新发布的内容”，形成一个**可持续更新的内容资源库**。

### 2. 核心特征

- **来源固定**
  - 来源在系统设计阶段就确定
  - 由人工调研、筛选得到
  - 数量有限、质量稳定
- **采集内容强调“新”**
  - 新发布的文章
  - 新更新的教程
  - 新的题解 / editorial
  - 新的视频 / 课程
- **不追求覆盖全面**
  - 不做全网搜索
  - 不尝试穷举所有相关内容
- **运行方式稳定、周期性**
  - 定期运行（如每日 / 每周）
  - 每次只产生少量增量数据

### 3. 输入与输出

**输入**

- 固定来源列表（站点 / 栏目 / 频道 / 博客）

**输出**

- 一条条“新内容记录”，例如：
  - 标题
  - 来源
  - 发布时间
  - 内容类型（教程 / 题解 / 视频等）
  - 简要摘要

### 4. 在算法平台中的作用

Pipeline 1 在平台中承担的是：

- **外部信息入口**
- **新内容信号源**
- **平台内容更新的触发器**

它的存在使平台能够回答：

> “最近算法学习领域有哪些值得关注的新内容？”

### 5. 示例

**示例 1**

- 某权威算法教程网站更新了一篇新的专题文章
- Pipeline 1 在定期扫描时发现并采集该内容

**示例 2**

- 某竞赛平台发布了新一场比赛的官方题解
- Pipeline 1 将其作为新内容记录入库

**示例 3**

- 某教学频道发布了一期新的算法讲解视频
- Pipeline 1 采集视频的元信息（标题、简介、链接）

### 6. 一句话总结 Pipeline 1

> **Pipeline 1 是“固定来源的新内容采集机制”，
> 负责让算法平台持续获得可靠的新外部内容。**

------

## Pipeline 2：知识点驱动的深度内容采集 Pipeline

(Topic-driven Deep Exploration Pipeline)

### 1. Pipeline 定义

**Pipeline 2 的目标是：**

> 围绕某一个或一组**明确的算法知识点**，系统性地采集大量相关学习资料，形成**按知识点组织的内容集合**。

### 2. 核心特征

- **由知识点驱动**
  - 每次运行都围绕一个具体主题
  - 例如某个算法、某类问题、某个技巧
- **来源不固定**
  - 不依赖预设的来源白名单
  - 允许来自不同网站、平台、作者
- **允许数据量大、冗余多**
  - 内容可能重复
  - 表述方式可能不同
- **运行频率低**
  - 不适合高频执行
  - 更偏向阶段性、按需触发

### 3. 输入与输出

**输入**

- 一个明确的算法知识点

**输出**

- 一个围绕该知识点的内容集合，例如：
  - 多篇教程 / 博客
  - 多种题解与解法
  - 不同讲解角度的材料
  - 新旧内容混合存在

### 4. 在算法平台中的作用

Pipeline 2 在平台中承担的是：

- **知识点扩展与补充**
- **深度学习资源积累**
- **为知识库净化提供素材**

它的存在使平台能够回答：

> “围绕某个算法知识点，我们目前能提供哪些学习资料？”

### 5. 注意

- Pipeline 2 **不是以新旧对比为目标运行**
- 新旧对比只是：
  - 在大量采集结果中
  - 事后被挑选出来的展示案例
- 其作用是：
  - 说明系统能够发现知识演化
  - 而不是限定爬取目标

### 6. 示例

**示例 1**

- 以某算法为主题，采集来自不同站点的讲解、题解和实现

**示例 2**

- 针对某一经典问题，收集不同年代、不同复杂度的解法资料

**示例 3**

- 汇总某知识点相关的教程、竞赛题和变体问题，形成完整资料池

### 7. 一句话总结 Pipeline 2

> **Pipeline 2 是“围绕具体知识点的深度内容采集机制”，
> 负责为算法平台构建可扩展的知识资料池。**

------

## 两条 Pipeline 的系统级对比

| 维度     | Pipeline 1   | Pipeline 2     |
| -------- | ------------ | -------------- |
| 核心驱动 | 来源         | 知识点         |
| 来源策略 | 固定、有限   | 不固定、开放   |
| 数据规模 | 小           | 大             |
| 更新频率 | 高频、稳定   | 低频、按需     |
| 内容角色 | 新内容信号   | 学习资料集合   |
| 平台作用 | 内容更新入口 | 知识库扩展支撑 |

------

## 总结

> **Pipeline 1 负责从固定可信来源中持续获取新内容，
> Pipeline 2 负责围绕具体算法知识点系统性地采集学习资料。
> 二者在数据形态上相似，但驱动方式、运行节奏和平台角色不同，因此需要作为两条独立 pipeline 设计。**

------

## 理解

比如我正在学习llm偏工程一些的内容，我会比较关注一些作者的主页博客、一些大厂的技术博客等，这些是我获取比较新的信息的固定来源；同时，我在找到我想深入的时候，我又会全网找一些除了这些博客等信息之外的学习资料来深入学习，也确实也可以代入上面两个pipeline
